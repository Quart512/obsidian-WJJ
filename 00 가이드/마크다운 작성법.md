# 마크다운 작성법

> [!tip] 마크다운이란?
> 마크다운은 **일반 텍스트로 서식을 표현하는 경량 마크업 언어**입니다. `#`, `*`, `[]` 같은 기호를 사용해 제목, 굵기, 링크 등을 표현합니다. 옵시디언은 표준 마크다운에 더해 자체적인 확장 문법도 지원합니다.

---

## 기본 마크다운 문법

### 헤딩 (제목)

```markdown
# 1단계 제목 (H1)
## 2단계 제목 (H2)
### 3단계 제목 (H3)
#### 4단계 제목 (H4)
##### 5단계 제목 (H5)
###### 6단계 제목 (H6)
```

> [!warning] 헤딩과 태그 구분
> `#제목` 처럼 `#` 바로 뒤에 텍스트를 붙이면 **태그**가 됩니다. 헤딩은 반드시 `# 제목` 처럼 공백이 있어야 합니다.

---

### 텍스트 강조

| 문법 | 결과 | 용도 |
|------|------|------|
| `**굵게**` | **굵게** | 중요한 개념, 키워드 |
| `*기울임*` 또는 `_기울임_` | *기울임* | 책 제목, 강조 |
| `***굵고 기울임***` | ***굵고 기울임*** | 매우 중요한 내용 |
| `~~취소선~~` | ~~취소선~~ | 삭제된 내용 |
| `==하이라이트==` | ==하이라이트== | 형광펜 효과 (옵시디언 전용) |
| `` `인라인 코드` `` | `인라인 코드` | 코드, 명령어, 파일명 |

### 인용문 (Blockquote)

```markdown
> 이것은 인용문입니다.
> 여러 줄도 가능합니다.

> [!note] 제목 있는 인용
> 옵시디언에서는 콜아웃으로 확장됩니다.
```

결과:
> 이것은 인용문입니다.
> 여러 줄도 가능합니다.

### 리스트

**순서 없는 리스트**
```markdown
- 항목 1
- 항목 2
  - 중첩 항목 (스페이스 2칸 또는 탭)
  - 중첩 항목 2
- 항목 3
```

**순서 있는 리스트**
```markdown
1. 첫 번째
2. 두 번째
3. 세 번째
   1. 중첩 순서 리스트
   2. 중첩 순서 리스트
```

**체크박스 (할일 목록)**
```markdown
- [ ] 아직 완료하지 않은 할일
- [x] 완료한 할일
- [-] 취소된 할일 (옵시디언 Tasks 플러그인)
```

### 코드 블록

**인라인 코드**: 백틱 하나로 감쌉니다.
```markdown
`print("Hello, World!")`
```

**여러 줄 코드 블록**: 백틱 세 개로 감싸고 언어를 지정합니다.

````markdown
```python
def hello():
    print("Hello, Obsidian!")
    return True
```
````

지원 언어 예시: `python`, `javascript`, `java`, `sql`, `bash`, `markdown`, `yaml`, `json`

### 표 (Table)

```markdown
| 열 1 | 열 2 | 열 3 |
|------|------|------|
| 내용 | 내용 | 내용 |
| 내용 | 내용 | 내용 |

| 왼쪽 정렬 | 가운데 정렬 | 오른쪽 정렬 |
|:----------|:-----------:|------------:|
| 텍스트    |   텍스트    |      텍스트 |
```

> [!tip] Advanced Tables 플러그인
> [[추천 플러그인]]에서 소개하는 **Advanced Tables** 플러그인을 설치하면 표를 훨씬 쉽게 만들고 편집할 수 있습니다. `Tab` 키로 셀 이동, 자동 정렬이 지원됩니다.

```markdown
---
```

### 링크와 이미지

**외부 링크**
```markdown
[링크 텍스트](https://obsidian.md)
```

**이미지**
```markdown
![대체 텍스트](이미지 경로 또는 URL)
```

**각주**
```markdown
이 문장에는 각주가 있습니다.[^1]

[^1]: 이것이 각주 내용입니다.
```

## 옵시디언 전용 문법

### 내부 링크 `[[]]`

옵시디언에서 가장 중요한 문법입니다.

```markdown
[[노트 이름]]                    → 노트로 이동하는 링크
[[노트 이름|표시할 텍스트]]       → 링크 텍스트를 다르게 표시
[[노트 이름#헤딩]]               → 특정 섹션으로 이동
[[노트 이름#헤딩|표시 텍스트]]   → 섹션 링크 + 커스텀 텍스트
```

예시:
```markdown
오늘 [[아토믹 해빗]]을 읽었다.
[[메모 관리 시스템#PARA 방법론|PARA 방법론]]에 대해 알아보자.
```

---

### 임베드 `![[]]`

다른 노트나 파일의 내용을 현재 노트 안에 **삽입**합니다.

```markdown
![[다른 노트]]                  → 노트 전체를 삽입
![[다른 노트#헤딩]]             → 특정 섹션만 삽입
![[이미지.png]]                 → 이미지 삽입
![[이미지.png|200]]             → 이미지 크기 지정 (픽셀)
![[문서.pdf]]                   → PDF 삽입
```

> [!example] 임베드 활용 예시
> MOC(목차 노트)에서 각 하위 노트의 첫 섹션을 임베드하면, 한 페이지에서 여러 노트의 핵심 내용을 볼 수 있습니다.

### 태그 `#`

```markdown
#태그이름
#상위태그/하위태그/세부태그
```

예시:
```markdown
#독서 #개발/파이썬 #프로젝트/진행중
```

태그는 노트 내용 어디에든 사용할 수 있고, YAML 프론트매터에도 넣을 수 있습니다.

```yaml
---
tags:
  - 독서
  - 개발/파이썬
---
```

---

### YAML 프론트매터

노트 맨 위에 `---` 으로 감싸서 메타데이터를 저장합니다.

```yaml
---
title: 노트 제목
created: 2026-03-22
tags:
  - 가이드
  - 마크다운
author: 홍길동
status: 진행중
rating: 5
---
```

> [!tip] Dataview와 함께 사용
> [[추천 플러그인]]의 **Dataview** 플러그인은 이 프론트매터 데이터를 기반으로 노트를 검색하고 표로 만들 수 있습니다.

### 콜아웃 (Callout)

인용문 블록을 확장한 옵시디언 전용 기능입니다. 중요한 정보를 시각적으로 강조할 때 사용합니다.

**기본 형식**
```markdown
> [!타입] 제목 (선택사항)
> 내용을 여기에 작성합니다.
```

**사용 가능한 콜아웃 타입**

```markdown
> [!note] 노트
> 일반적인 참고 사항

> [!tip] 팁
> 유용한 정보나 요령

> [!important] 중요
> 반드시 알아야 할 정보

> [!warning] 경고
> 주의가 필요한 내용

> [!danger] 위험
> 오류나 위험한 동작

> [!success] 성공
> 잘된 예시나 완료 표시

> [!question] 질문
> 미해결 질문이나 고민

> [!example] 예시
> 구체적인 예시

> [!quote] 인용
> 책이나 다른 출처의 인용

> [!abstract] 요약
> 내용 요약
```

**접을 수 있는 콜아웃**
```markdown
> [!tip]- 클릭해서 펼쳐보세요
> 처음에는 접혀 있는 콜아웃입니다.

> [!tip]+ 기본으로 펼쳐진 접기 가능 콜아웃
> 클릭하면 접을 수 있습니다.
```

### 수식 (LaTeX)

```markdown
인라인 수식: $E = mc^2$

블록 수식:
$$
\sum_{i=1}^{n} x_i = \frac{n(n+1)}{2}
$$
```

> [!note] 수식 렌더링
> 수식은 설정에서 활성화해야 렌더링됩니다. **설정 > 편집기 > 수식 렌더링** 을 켜세요.

## 마크다운 작성 팁

> [!tip] 실시간 미리보기 활용
> 옵시디언은 기본적으로 **실시간 미리보기(Live Preview)** 모드를 사용합니다. 문법을 입력하면 즉시 렌더링 결과를 볼 수 있습니다. `Ctrl/Cmd + E` 로 소스 모드와 미리보기 모드를 전환할 수 있습니다.

> [!example] 좋은 노트 작성 습관
> - 헤딩 계층을 논리적으로 구성하세요 (H1 → H2 → H3)
> - 핵심 단어는 **굵게** 표시하세요
> - 긴 문단보다 짧은 문단과 리스트를 활용하세요
> - 관련 노트는 항상 `[[내부링크]]` 로 연결하세요
