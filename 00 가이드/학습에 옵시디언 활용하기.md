# 학습에 옵시디언 활용하기

> [!tip] 이 가이드의 목적
> 단순히 정보를 저장하는 것을 넘어, 옵시디언을 활용해 **진짜 배우고 기억에 남는 학습 시스템**을 만드는 방법을 알아봅니다.

---

## 독서 노트 작성법

### 왜 독서 노트가 중요한가?

책을 읽는 것만으로는 지식이 남지 않습니다. 연구에 따르면 아무런 처리 없이 읽은 내용은 며칠 안에 80% 이상 잊어버립니다. 독서 노트는 정보를 **장기 기억으로 전환**하는 과정입니다.

### 독서 노트 작성 프로세스

**1단계: 읽으면서 (플리팅 메모)**
- 중요하다고 생각되는 부분에 하이라이트
- 간단한 메모를 여백에 (또는 임시 노트에)
- "왜 이게 중요한가?" "어디에 적용할 수 있을까?"를 자문

**2단계: 읽은 후 (독서 노트)**
- 책 전체에 대한 노트 하나 생성
- [[템플릿 모음#독서 노트 템플릿|독서 노트 템플릿]] 사용 권장

**3단계: 며칠 후 (영구 메모)**
- 독서 노트를 다시 읽으며 핵심 아이디어 추출
- 각 아이디어를 별도의 원자적 노트로 작성
- 기존 노트들과 연결

### 좋은 독서 노트의 구조

```markdown
---
title: 아토믹 해빗
author: 제임스 클리어
published: 2018
read: 2026-03-22
rating: 5
tags:
  - 유형/독서노트
  - 주제/습관
  - 주제/생산성
---

# 아토믹 해빗

## 한 줄 요약
작은 습관의 1% 개선이 복리처럼 쌓여 큰 변화를 만든다.

## 핵심 아이디어
- [[습관의 복리 효과]] - 1% 개선이 1년 후 37배
- [[정체성 기반 습관]] - "나는 ~하는 사람이다"
- [[습관 루프의 4단계]] - 신호, 갈망, 반응, 보상
- [[2분 규칙]] - 새 습관은 2분 이내로 시작

## 기억에 남는 구절
> "당신은 목표 수준으로 오르는 것이 아니라, 시스템 수준으로 떨어진다." (p.27)

## 나에게 적용할 것
- [ ] 아침 독서 습관 만들기 (2분 규칙 적용)
- [ ] 습관 트래커 만들기

## 관련 책
- [[원자 단위의 삶]] (관련 주제)
- [[습관의 힘 - 찰스 두히그]] (같은 주제 다른 시각)
```

## 강의/영상 노트 작성법

### Cornell 노트 방식

강의를 들을 때 효과적인 **Cornell 노트** 방식을 옵시디언에 적용합니다.

```markdown
---
source: "[강의명] - 강사명"
date: 2026-03-22
tags:
  - 유형/강의노트
---

# [강의명] - 3강: 주제

## 핵심 질문
(강의를 들으면서 생기는 질문들)
- 왜 X는 Y보다 효율적인가?
- Z를 실제로 어떻게 적용하는가?

## 노트
(강의 내용 요약)
### 주요 개념 A
- 내용...
- 내용...

### 주요 개념 B
- 내용...

## 요약
(강의 후 5~10줄로 핵심 정리)
이 강의에서 배운 핵심은...

## 다음 행동
- [ ] 예제 코드 직접 작성해보기
- [ ] 관련 문서 찾아보기
```

### 영상 노트 팁

> [!tip] 영상을 보면서 노트하는 방법
> 1. 처음에는 그냥 전체를 봅니다
> 2. 다시 볼 때 타임스탬프와 함께 핵심만 적습니다
> 3. `[12:34]` 형식으로 타임스탬프를 기록하면 나중에 찾기 편합니다
>
> ```markdown
> [03:22] 습관 루프의 4단계 설명
> [15:40] 실제 적용 사례 - 운동 습관
> [28:15] 환경 설계의 중요성
> ```

## 점진적 요약 기법 (Progressive Summarization)

Tiago Forte가 개발한 기법으로, 노트를 여러 레이어로 점점 요약해 나가는 방법입니다.

### 5단계 레이어

**레이어 0: 원본 저장**
아티클, 책의 발췌, 강의 내용을 그대로 저장

**레이어 1: 첫 하이라이트**
처음 읽을 때 중요하다고 생각되는 부분을 **굵게** 표시

```markdown
원래 텍스트에서 중요한 부분에 **굵게** 처리합니다.
모든 것을 강조하지 말고 정말 핵심적인 10~20%만 선택하세요.
```

**레이어 2: 두 번째 하이라이트**
며칠 후 다시 읽을 때, 레이어 1에서 가장 중요한 부분을 ==하이라이트==

```markdown
레이어 1에서 이미 굵게 표시된 내용 중
==정말 핵심인 5~10%를 하이라이트==합니다.
```

**레이어 3: 미니 요약**
노트 맨 위에 3~5문장의 요약 작성

```markdown
## 미니 요약 (레이어 3)
이 글의 핵심 주장은 X이다. Y를 통해 Z를 달성할 수 있다.
실제 적용 방법은 A, B, C이다.
```

**레이어 4: 재창조**
완전히 자신의 말로 새로운 노트 작성

> [!important] 모든 노트를 레이어 4까지 만들 필요는 없습니다
> 레이어 4는 정말 중요하고 자주 참조하는 내용에만 적용하세요. 대부분의 노트는 레이어 2~3으로 충분합니다.

## 플래시카드와 반복 학습

### 옵시디언에서 플래시카드 만들기

**Spaced Repetition 플러그인**을 사용하거나, 노트 안에 Q&A 형식으로 작성합니다.

**간단한 Q&A 형식**
```markdown
## 복습 카드

**Q: 습관 루프의 4단계는?**
A: 신호 → 갈망 → 반응 → 보상

**Q: 2분 규칙이란?**
A: 새로운 습관을 시작할 때 2분 이내로 끝낼 수 있는 것부터 시작하는 방법

**Q: 정체성 기반 습관이란?**
A: "나는 ~를 하고 싶다"가 아닌 "나는 ~하는 사람이다"라는 정체성에서 습관을 시작하는 접근법
```

**Anki 연동 (고급)**

[[추천 플러그인]]에서 **Obsidian to Anki** 플러그인을 설치하면, 옵시디언 노트의 Q&A를 Anki 카드로 자동 내보낼 수 있습니다.

### 반복 학습 스케줄

Dataview로 복습할 노트 관리:
```yaml
---
last_reviewed: 2026-03-15
next_review: 2026-03-22
review_count: 3
---
```

````markdown
```dataview
TABLE next_review, review_count
FROM #복습필요
WHERE next_review <= date(today)
SORT next_review ASC
```
````

## 프로젝트와 연결하기

### 지식을 프로젝트에 활용하는 흐름

```
자료 수집 → 노트 정리 → 링크 연결 → 프로젝트에 인용
```

**프로젝트 노트에서 관련 자료 링크**

```markdown
# 마케팅 캠페인 기획

## 참고한 자료
- [[행동 경제학의 넛지 이론]] - 사용자 행동 유도 방법
- [[카피라이팅 기법 정리]] - 문구 작성에 활용
- [[경쟁사 분석 2026]] - 시장 현황 파악

## 적용할 인사이트
- [[손실 회피 편향]]을 활용한 한정 오퍼 문구
```

**관련 자료 Dataview로 자동 수집**

````markdown
```dataview
LIST
FROM #주제/마케팅
WHERE file.name != "마케팅 캠페인 기획"
SORT file.mtime DESC
LIMIT 10
```
````

## 지식 베이스 구축 팁

### 1. 매일 조금씩

> [!tip] 완벽한 노트보다 불완전한 노트
> 매일 5~10분이라도 노트를 쓰는 것이 가끔 완벽한 노트를 쓰는 것보다 훨씬 효과적입니다. "지금 당장 완성 못해도 괜찮다. 나중에 추가하면 된다."는 마음으로 시작하세요.

### 2. 연결 우선

새 노트를 쓸 때마다 **"이 아이디어와 연결할 수 있는 기존 노트가 있는가?"** 를 반드시 생각하세요. 연결이 없는 노트는 고립된 섬처럼 쓸모가 없어집니다.

### 3. 자신의 말로 재작성

> [!important] 복사-붙여넣기 금지
> 책이나 기사의 내용을 그대로 복사하면 이해한 것이 아닙니다. 항상 **자신의 말로 재작성**하세요. "내가 이것을 친구에게 설명한다면?" 하고 생각하며 쓰세요.

### 4. 질문을 노트 제목으로

명사형 제목 대신 질문형 제목을 사용하면 글을 더 능동적으로 쓸 수 있습니다.

```
나쁜 예: "습관"
좋은 예: "좋은 습관을 만들려면 어떻게 해야 할까?"

나쁜 예: "인공지능"
좋은 예: "AI는 창의적 직업을 대체할 수 있을까?"
```

### 5. 정기적인 리뷰

- **주간 리뷰**: 이번 주 만든 노트 훑어보기, 연결 추가
- **월간 리뷰**: 자주 접근하는 노트/주제 파악, 새 MOC 만들기
- **분기 리뷰**: 전체 볼트 구조 점검, 보관 폴더로 이동

### 6. 그래프 뷰 활용

그래프 뷰(`Ctrl/Cmd + G`)를 주기적으로 보세요. **연결이 없는 고립된 노트**들을 찾아서 연결해주면 볼트가 점점 강력해집니다.

> [!success] 지식 베이스 성장의 증거
> 처음에는 그래프 뷰가 듬성듬성 보입니다. 3개월 후, 6개월 후 점점 촘촘해지는 그래프를 보는 것이 가장 큰 동기부여가 됩니다.

## 학습 워크플로우 요약

```
오늘 배운 것
    ↓
일일 노트에 빠르게 기록 (플리팅 메모)
    ↓
저녁 또는 주말에 정식 노트로 정리
    ↓
자신의 말로 재작성
    ↓
기존 노트와 링크 연결
    ↓
관련 프로젝트에 활용
    ↓
주기적 복습
```
