## 🏗️ UE5 아키텍처 개요

### 핵심 클래스 계층
```
UObjectBase
└── UObject
    ├── AActor (월드에 존재하는 모든 객체)
    │   ├── APawn (플레이어/적)
    │   │   └── ACharacter (폰 + 이동 컴포넌트)
    │   ├── AInfo (데이터 저장)
    │   └── AController (행동 제어)
    │       ├── APlayerController
    │       └── AAIController
    └── UActorComponent (컴포넌트)
        ├── USceneComponent (위치/회전)
        └── UActorComponent (로직)
```

## 🎮 게임별 클래스 구조

### 1. 플레이어 시스템

#### APlayerCharacter
```cpp
// 플레이어 캐릭터
class APlayerCharacter : public ACharacter
{
    // 컴포넌트
    UPROPERTY(VisibleAnywhere)
    class USpringArmComponent* CameraBoom;
    
    UPROPERTY(VisibleAnywhere)
    class UCameraComponent* FollowCamera;
    
    UPROPERTY(VisibleAnywhere)
    class UHealthComponent* HealthComponent;
    
    UPROPERTY(VisibleAnywhere)
    class UInventoryComponent* InventoryComponent;
    
    // 함수
    virtual void BeginPlay() override;
    virtual void Tick(float DeltaTime) override;
    
    void MoveForward(float Value);
    void MoveRight(float Value);
    void Jump();
    void Attack();
};
```

#### APlayerController
```cpp
// 플레이어 컨트롤러
class AMyPlayerController : public APlayerController
{
    virtual void BeginPlay() override;
    virtual void SetupInputComponent() override;
    
    // 입력 바인딩
    void MoveForward(float Value);
    void MoveRight(float Value);
    void Look(float Value);
};
```

### 2. 적 시스템

#### AEnemyBase
```cpp
// 적 기본 클래스
class AEnemyBase : public ACharacter
{
    UPROPERTY(VisibleAnywhere)
    class UHealthComponent* HealthComponent;
    
    UPROPERTY(VisibleAnywhere)
    class UAIPerceptionComponent* AIPerception;
    
    // 상태 머신
    enum class EEnemyState
    {
        Idle,
        Patrol,
        Chase,
        Attack,
        Dead
    };
    
    EEnemyState CurrentState;
    
    virtual void Tick(float DeltaTime) override;
    void UpdateState(EEnemyState NewState);
};
```

#### AEnemyAIController
```cpp
// 적 AI 컨트롤러
class AEnemyAIController : public AAIController
{
    virtual void BeginPlay() override;
    
    // 행동 트리
    UPROPERTY(EditDefaultsOnly)
    class UBehaviorTree* BehaviorTree;
};
```

## 🔧 커스텀 컴포넌트

### UHealthComponent
```cpp
// 체력 컴포넌트
UCLASS()
class UHealthComponent : public UActorComponent
{
    GENERATED_BODY()
    
    UPROPERTY(EditDefaultsOnly, Category = "Health")
    float MaxHealth;
    
    UPROPERTY(VisibleAnywhere, Category = "Health")
    float CurrentHealth;
    
    // 델리게이트
    UPROPERTY(BlueprintAssignable)
    FOnDeathDelegate OnDeath;
    
    UPROPERTY(BlueprintAssignable)
    FOnHealthChangedDelegate OnHealthChanged;
    
public:
    void TakeDamage(float Damage);
    void Heal(float Amount);
    bool IsDead() const;
};
```

## 🎯 설계 원칙

### 1. SOLID 원칙
- **Single Responsibility**: 한 클래스 = 한 역할
- **Open/Closed**: 확장은 열림, 수정은 닫힘
- **Liskov Substitution**: 자식은 부모 대체 가능
- **Interface Segregation**: 인터페이스 분리
- **Dependency Inversion**: 추상화 의존

### 2. UE5 모범 사례
- **컴포넌트 기반 설계**
- **블루프린트 친화적**
- **리플리케이션 고려**
- **메모리 관리**
