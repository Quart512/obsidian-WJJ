# 메모 관리 시스템

> [!note] PKM이란?
> **PKM(Personal Knowledge Management)** 은 개인 지식 관리를 뜻합니다. 정보를 수집하고, 정제하고, 연결하고, 활용하는 개인적인 시스템입니다. 옵시디언은 PKM을 위한 최고의 도구 중 하나입니다.

---

## PARA 방법론

Tiago Forte가 개발한 **PARA(Projects, Areas, Resources, Archives)** 는 폴더 기반으로 정보를 관리하는 가장 실용적인 방법론입니다.

### PARA의 4가지 카테고리

```
볼트/
├── 01 프로젝트 (Projects)    ← 현재 진행 중인 목표
├── 02 영역 (Areas)           ← 지속적으로 관리하는 책임 영역
├── 03 자료 (Resources)       ← 관심 주제, 참고 자료
└── 04 보관 (Archives)        ← 완료되거나 비활성화된 항목
```

### Projects (프로젝트)

**정의**: 명확한 완료 시점이 있는 현재 진행 중인 작업

```
01 프로젝트/
├── 2026-Q1 마케팅 캠페인/
├── 신제품 기획서/
├── 블로그 포스팅 시리즈/
└── 자격증 취득 공부/
```

특징:
- 완료되면 **보관(Archives)** 으로 이동
- 각 프로젝트는 명확한 목표와 마감일이 있음
- 현재 집중하고 있는 것들만 유지 (너무 많으면 안됨)

### Areas (영역)

**정의**: 지속적으로 유지해야 하는 책임이나 관심 분야

```
02 영역/
├── 건강 & 운동/
├── 재정 관리/
├── 커리어 개발/
├── 인간 관계/
└── 개인 성장/
```

특징:
- 완료 시점이 없음 (지속적으로 관리)
- 개인적인 역할과 책임 영역
- 프로젝트는 생겨나고 사라지지만, 영역은 비교적 안정적

### Resources (자료)

**정의**: 미래에 유용할 수 있는 주제별 참고 자료

```
03 자료/
├── 독서 노트/
├── 개발/
│   ├── Python/
│   └── 웹 개발/
├── 디자인/
├── 마케팅/
└── 생산성/
```

특징:
- 현재 활성 프로젝트와 직접 관련 없어도 됨
- 관심 있는 주제라면 무엇이든
- 나중에 프로젝트에 활용될 수 있는 지식 저장소

### Archives (보관)

**정의**: 완료된 프로젝트, 더 이상 관심 없는 자료

```
04 보관/
├── 완료된 프로젝트/
│   ├── 2025-12 연말 보고서/
│   └── 2025-09 사이드 프로젝트/
└── 과거 자료/
```

> [!tip] PARA 활용 팁
> 완료된 프로젝트는 삭제하지 말고 보관 폴더로 이동하세요. 나중에 참고할 일이 생깁니다. "이 정보는 어디에 두어야 하지?"가 아니라 "이 정보는 언제 필요할까?"를 기준으로 분류하세요.

## Zettelkasten 방법론

독일 사회학자 **Niklas Luhmann**이 개발한 지식 연결 시스템입니다. 그는 이 방법으로 70권 이상의 책과 400편 이상의 논문을 썼습니다.

### Zettelkasten의 핵심 원칙

**1. 원자적 노트 (Atomic Notes)**
각 노트는 **하나의 아이디어**만 담아야 합니다. 여러 아이디어를 한 노트에 담지 말고, 각각 별도의 노트로 만드세요.

```
나쁜 예: "독서 정리 - 아토믹 해빗"
(책 전체 내용이 하나의 파일)

좋은 예:
"습관의 복리 효과"
"정체성 기반 습관의 힘"
"2분 규칙"
(각 핵심 아이디어가 별도 파일)
```

**2. 영구 메모 (Permanent Notes)**
세 가지 종류의 메모를 사용합니다:

| 종류 | 설명 | 예시 |
|------|------|------|
| 플리팅 메모 (Fleeting) | 빠르게 적는 임시 메모 | 아이디어, 메모, 스크랩 |
| 문헌 메모 (Literature) | 특정 출처에서 정리한 내용 | 독서 노트, 강의 노트 |
| 영구 메모 (Permanent) | 자신의 말로 재작성한 핵심 아이디어 | 최종 지식 노트 |

**3. 링크로 연결**
노트를 폴더가 아닌 **링크**로 연결합니다. 하나의 아이디어는 여러 맥락에서 동시에 참조될 수 있습니다.

**4. 고유 ID**
각 노트에 고유한 ID를 부여합니다 (옵시디언에서는 타임스탬프 형식이 일반적):

```
202603221430 - 습관의 복리 효과.md
202603221445 - 정체성 기반 습관의 힘.md
```

### Zettelkasten 워크플로우

```
읽기/경험
    ↓
플리팅 메모 (빠르게 기록)
    ↓
문헌 메모 (출처 정리)
    ↓
영구 메모 (자신의 언어로 재작성)
    ↓
기존 메모와 링크 연결
    ↓
새로운 통찰 발생
```

> [!example] 실제 예시
> 1. 책을 읽다가 "습관은 복리처럼 쌓인다"는 구절을 발견
> 2. 플리팅 메모: "아토믹 해빗 - 작은 습관도 꾸준히 하면 큰 결과 - p.16"
> 3. 문헌 메모: 책의 관련 내용을 정리
> 4. 영구 메모: "습관의 복리 효과" 노트를 만들고, 이미 있는 "복리의 힘", "장기적 사고" 노트와 연결

## MOC (Map of Content) 활용법

**MOC**는 관련된 노트들을 모아두는 **허브 노트**입니다. 폴더의 대안으로, 하나의 노트가 여러 MOC에 포함될 수 있습니다.

### MOC의 역할

```
MOC: Python 프로그래밍
├── [[변수와 자료형]]
├── [[함수와 클래스]]
├── [[파일 입출력]]
├── [[예외 처리]]
└── [[주요 라이브러리]]
    ├── [[NumPy 기초]]
    └── [[Pandas 데이터 분석]]
```

### MOC 작성 방법

```markdown
# Python 프로그래밍 MOC

## 기초 문법
- [[변수와 자료형]]
- [[조건문과 반복문]]
- [[함수 정의와 사용]]

## 객체지향
- [[클래스와 객체]]
- [[상속과 다형성]]

## 유용한 라이브러리
- [[NumPy로 수치 계산]]
- [[Pandas로 데이터 분석]]
- [[Matplotlib 시각화]]

## 관련 MOC
- [[프로그래밍 언어 MOC]]
- [[데이터 사이언스 MOC]]
```

> [!tip] MOC 시작 시점
> 특정 주제에 대한 노트가 5~10개 이상 쌓이기 시작하면 MOC를 만들 때입니다. 처음부터 MOC를 만들 필요는 없습니다.

## 폴더 구조 예시와 추천 구조

### 초보자 추천 구조 (단순하게 시작)

```
볼트/
├── 00 가이드/          ← 옵시디언 사용법 (지금 이 폴더)
├── 01 프로젝트/        ← 진행 중인 프로젝트
├── 02 영역/            ← 지속 관리 영역
├── 03 자료/            ← 참고 자료와 독서 노트
├── 04 보관/            ← 완료된 것들
├── 99 템플릿/          ← 노트 템플릿 모음
├── Daily/              ← 일일 노트
└── 홈.md              ← 볼트 메인 허브
```

### 학습 중심 구조

```
볼트/
├── 00 가이드/
├── 10 플리팅 메모/     ← 즉석 아이디어
├── 20 문헌 메모/       ← 독서/강의 노트
├── 30 영구 메모/       ← 핵심 지식 노트
├── 40 프로젝트/        ← 진행 중인 프로젝트
├── 50 MOC/             ← 주제별 허브 노트
├── Daily/
├── 99 템플릿/
└── 홈.md
```

### 업무/직장인 중심 구조

```
볼트/
├── 00 가이드/
├── 01 현재 프로젝트/
├── 02 업무 영역/
│   ├── 회의록/
│   ├── 1:1 미팅/
│   └── 팀 정보/
├── 03 학습/
├── 04 개인/
├── 04 보관/
├── Daily/
├── 99 템플릿/
└── 홈.md
```

> [!important] 완벽한 구조는 없습니다
> 폴더 구조에 너무 많은 시간을 쓰지 마세요. 처음에는 단순하게 시작하고, 사용하면서 자연스럽게 발전시키는 게 최선입니다. 폴더보다 **링크와 태그**가 훨씬 강력한 조직 도구입니다.

## 태그 전략

### 태그 vs 폴더 vs 링크

| 방법 | 장점 | 단점 |
|------|------|------|
| 폴더 | 직관적, 외부 앱에서도 인식 | 하나의 위치만 가능 |
| 태그 | 여러 카테고리 동시 지정 | 관리하면 복잡해짐 |
| 내부 링크 | 맥락 있는 연결, 양방향 | 명시적으로 연결해야 함 |

세 가지를 **조합**해서 사용하는 것이 최선입니다.

### 효과적인 태그 시스템 예시

**상태 태그**
```
#상태/진행중
#상태/완료
#상태/보류
#상태/초안
```

**유형 태그**
```
#유형/독서노트
#유형/프로젝트
#유형/회의록
#유형/아이디어
#유형/참고자료
```

**주제 태그**
```
#주제/개발/파이썬
#주제/개발/웹
#주제/경제/투자
#주제/건강/운동
```

**중요도 태그**
```
#중요
#나중에검토
#공유가능
```

### 태그 사용 팁

> [!tip] 태그는 적을수록 좋습니다
> 너무 많은 태그를 만들면 관리가 어려워집니다. 태그는 **검색할 때 실제로 쓸 것들**만 만드세요. "이 태그로 나중에 검색할 것인가?"를 자문해보세요.

Dataview로 태그별 노트 조회:
````markdown
```dataview
TABLE file.name, file.ctime
FROM #유형/독서노트
SORT file.ctime DESC
```
````

## 어떤 방법론을 선택할까?

> [!question] 나에게 맞는 방법론은?
> - **직장인/프리랜서** → PARA 방법론 추천 (프로젝트 중심)
> - **학생/연구자** → Zettelkasten 추천 (지식 구축 중심)
> - **둘 다 필요** → PARA 폴더 구조 + Zettelkasten 노트 작성 방식 혼용
>
> 처음에는 **PARA로 폴더 구조**를 잡고, **링크와 태그로 연결**하는 것을 추천합니다.
